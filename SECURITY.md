# Beveiliging

Deze richtlijn geldt voor **alle repositories onder [security-commons-nl](https://github.com/security-commons-nl)**. Individuele repos hoeven geen eigen SECURITY.md te onderhouden.

## Kwetsbaarheid melden

Ontdek je een beveiligingsprobleem? Meld het **niet** via een openbaar issue of discussion — dat zou andere gebruikers in de tussentijd blootstellen.

### Hoe meld je het?

Gebruik GitHub's **Private Vulnerability Reporting**:

1. Ga naar de **Security**-tab van de betreffende repository
2. Klik op *"Report a vulnerability"*
3. Vul het formulier in — alleen zichtbaar voor maintainers

Als de Security-tab geen reporting-optie toont, stuur een directe bericht via [LinkedIn naar de maintainer](https://linkedin.com/in/bas-stevens).

### Wat kun je verwachten?

- **Ontvangstbevestiging** binnen 5 werkdagen
- **Tussentijdse updates** tijdens het onderzoek
- **Responsible-disclosure-timeline** — standaard 90 dagen, in overleg aan te passen
- **GitHub Security Advisory** publiceren we na de fix
- **Credit** voor jou als ontdekker (indien gewenst)

## Scope

Alle code en artefacten in repositories onder `github.com/security-commons-nl`.

**Buiten scope**: externe diensten die onze tools hosten of consumeren (GitHub, Mistral, Ollama, Scaleway, etc.). Meld die via hun eigen kanalen.

## Geen bug bounty

Dit is een publieke commons zonder commerciële belangen. We betalen geen bounty, maar waarderen meldingen met vermelding in de advisory en release-notes.
